# BurstPick 沉浸式选片界面改造设计

## 目标

在保留现有扫描、连拍分组、评分持久化、Lightroom 元数据写入和复制导出能力的基础上，重做日常选片交互。新版界面必须让照片成为绝对主体，减少用户理解“相册、分组、焦点、多选、导出类型”等内部概念的成本，并让连续查看、放大和评分形成不间断的键盘工作流。

成功标准是：用户选择照片目录后直接进入第一组，默认看到足够大的照片；可以控制每屏显示数量，逐张独立评分，连续推进到下一批和下一组，最后通过单一“完成选片”入口把星级交给 Lightroom 或复制入选文件。

## 已确认的产品决策

- 采用沉浸暗色视觉方向，工具栏和辅助信息尽量隐退，避免影响照片颜色判断。
- 主工作台采用顺序浏览，不使用照片 PK 或淘汰赛逻辑。
- 每屏可显示 1、2、3 或 4 张，默认恢复用户上次选择。
- 多张照片仅并排展示；每张有独立焦点、独立星级和独立操作。
- 默认在评分后把焦点移到下一张；用户可以在设置中关闭自动移动。
- 焦点越过本屏最后一张时自动加载下一批，不停留或在本屏循环。
- 按住空格时临时以 100% 查看当前照片，松开立即恢复原布局，不打开或关闭模态框。
- 高级分组工具默认收起；导出由单一“完成选片”入口引导。

## 信息架构

应用保留三个清晰阶段：

1. **选择照片**：首页只突出“选择照片文件夹”和“体验示例”。手动路径和其他诊断能力归入次要区域。
2. **连续选片**：选择目录后直接打开第一组和第一张照片。照片、焦点、星级、组内进度和组导航构成主界面。
3. **完成选片**：用户点击唯一主操作“完成选片”，再选择写入 Lightroom 星级或复制入选文件。

扫描可以在后台继续，但首批照片可用后应立即进入选片界面。界面必须明确区分“可以开始选片，仍在后台分析”和“无法继续，需要重试”。

## 工作台布局

工作台使用沉浸暗色背景，由四个弱化区域组成：

- 顶栏：当前组、组内范围、已评分数量、每屏张数和“完成选片”。
- 照片舞台：占据剩余可用空间，按 1–4 张自动布局，保持照片比例，不裁掉主体。
- 照片信息：每张照片下方只显示安全文件名、星级和必要警告。
- 胶片带：显示当前连拍组的整体位置和当前可见范围，可点击跳转。

每屏一张时照片最大化显示。两张时采用等宽并排。三张和四张根据窗口比例选择单行或紧凑网格，但必须优先保证单张可判断性。窄屏自动降级为更少列，不产生横向滚动。

当前焦点使用克制但清晰的高亮边界，并辅以文本状态，不能只依赖颜色。非焦点照片仍可直接点击获得焦点。布局密度选择器是一个紧凑的 `1 / 2 / 3 / 4` 控件，变化立即生效并保存到本地偏好。

## 连续评分与导航

- 左右方向键按照片顺序移动焦点。
- `1`–`5` 设置当前照片星级，`0` 清除星级。
- 自动移动开启时，成功保存评分后焦点前进一张；关闭时留在当前照片。
- 焦点越过当前批次末尾时，下一批替换舞台内容，并把焦点放在该批第一张。
- 当前组末尾继续前进时自动进入下一组；必须显示轻量的组切换反馈。
- 向左移动遵循完全对称的规则，可以返回上一批或上一组。
- 点击胶片带缩略图直接把对应照片设为焦点，并让包含它的批次进入舞台。
- 所有移动都保持稳定照片顺序，不因评分或筛选改变位置。

评分持久化失败时不前进，并在当前照片附近显示可恢复错误。成功后不得用阻塞 toast 打断连续操作。

## 临时 100% 查看

空格键改为按住查看，而不是打开模态框：

- 按下空格时，当前焦点照片临时覆盖照片舞台，以 100% 比例显示。
- 松开空格后立刻恢复原来的 1–4 张布局、焦点和滚动位置。
- 临时查看期间可用鼠标或触控板平移大图；不改变评分或选择。
- 键盘重复事件不得重复创建查看层。
- 窗口失焦、标签页隐藏或组件卸载时必须自动退出临时查看，避免界面卡在放大状态。
- 文本输入、范围输入和其他表单控件聚焦时，空格键保持原生行为。

触屏设备没有“按住空格”，因此保留一个低强调的“查看 100%”按钮；按住按钮显示，松开或取消触摸恢复。

## 高级工具与完成流程

拆分组、合并组、重新分组和灵敏度属于高级工具，默认折叠在“调整分组”中。用户不需要理解这些功能即可完成整个选片流程。展开后仍沿用现有安全命令和撤销能力。

工作台只保留一个主要完成动作。点击“完成选片”后进入总结页：

- 显示总照片数、已评分数和至少一星的入选数量。
- 提供“写入 Lightroom 星级”和“复制入选照片”两个清晰选择。
- 元数据写入继续要求 dry run、冲突检查和 Lightroom 关闭确认。
- 复制导出继续要求选择目标目录、冲突预览和不覆盖保证。
- 高风险说明只在对应流程出现，不常驻选片工作台。

## 状态与偏好

新增的界面偏好仅保存在当前浏览器本地，不进入照片源目录：

- `photosPerPage`: `1 | 2 | 3 | 4`
- `advanceAfterRating`: `boolean`，默认 `true`
- `groupToolsExpanded`: `boolean`，默认 `false`

照片评分、分组和撤销历史仍由服务端会话持久化，浏览器偏好不能成为评分真相来源。恢复相册时，先恢复服务端照片状态，再应用显示偏好。

## 组件边界

- `CullingWorkspace`：组织顶栏、舞台、胶片带和完成入口。
- `PhotoStage`：根据每屏张数计算当前批次和响应式布局。
- `CullingPhoto`：展示单张照片、焦点、星级和安全警告。
- `Filmstrip`：展示组内位置和当前可见范围。
- `HoldToInspect`：管理空格按下/松开、100% 图层、平移和强制清理。
- `CullingPreferences`：读取、验证和保存界面偏好。
- `CompletionFlow`：承载选片总结以及元数据/复制导出的分流。
- `AdvancedGroupingTools`：包裹现有拆分、合并、灵敏度和重新分组操作。

组件继续通过现有 album hook 和 API 操作评分、导航及导出，不复制服务端业务逻辑。

## 可访问性与响应式要求

- 所有主要动作可通过键盘和指针完成。
- 焦点状态同时使用边界和可访问文本表达。
- 每张照片提供文件名、位置、评分和配对状态的可访问名称。
- 每屏张数控件使用原生按钮或单选语义，明确暴露当前值。
- 临时 100% 查看层正确标注当前照片，但不形成难以退出的焦点陷阱。
- 可见控件继续满足至少 44×44 px 的触控目标。
- 320px 宽度下不出现横向溢出；必要时自动减少舞台列数，但不修改用户保存的偏好值。
- 遵循 `prefers-reduced-motion`，批次和组切换不依赖动画传达状态。

## 测试策略

### 单元与组件测试

- 每屏 1–4 张的批次计算、边界前进和对称后退。
- 每屏张数偏好的验证、默认值、保存和恢复。
- 自动移动开启与关闭时的评分焦点行为。
- 评分失败时焦点不移动，错误保持在当前照片。
- 胶片带跳转到正确批次和照片。
- 空格按下、键盘重复、松开、窗口失焦和卸载清理。
- 表单控件聚焦时不拦截空格和数字键。
- 高级工具默认折叠，现有分组命令仍可访问。
- 完成页正确显示统计并进入现有两个导出流程。

### 浏览器验收

- 桌面视口下分别切换 1、2、3、4 张并验证独立评分。
- 两张模式中给两张照片不同星级，刷新后保持。
- 评分后自动跨批次和跨组前进；关闭设置后焦点停留。
- 按住空格进入 100%，松开恢复完全相同的批次和焦点。
- 胶片带跳转、完成选片、元数据预览和复制预览都可完成。
- 移动视口无横向溢出、控件尺寸合格、控制台无错误。

### 回归门

现有配对、扫描、分组、评分持久化、撤销、元数据事务、复制事务和安全测试必须保持通过。最终发布门仍为：测试、类型检查、lint、生产构建、Playwright E2E 和元数据安全 smoke 全部通过。

## 非目标

本次改造不增加自动审美评分、闭眼检测、人脸识别、照片删除、RAW 调色、Lightroom 目录库写入或桌面安装包。也不改变现有服务端评分和导出安全模型。
