# BurstPick iOS 风格设置与提交体验设计

## 目标

将现有沉浸式选片页重构为接近 iOS Photos 与系统设置的视觉和交互语言，让用户能立即找到设置、理解评分保存状态，并明确完成评分后的提交路径。保留现有本地优先、安全导出和即时评分持久化模型。

## 产品决策

- 评分继续即时保存到本地会话，不增加容易遗忘的“保存评分”步骤。
- 顶栏显示“正在保存 / 已保存 / 保存失败”，让即时保存可感知。
- “完成”是唯一的最终出口；它打开结果 Sheet，提供写入 Lightroom 与复制导出。
- 右上角齿轮打开 iOS 风格设置 Sheet。桌面端为居中的紧凑浮层，窄屏从底部升起。
- 每屏数量与评分后自动前进移入设置，不再挤占照片顶栏。
- 聚合阈值改名为“连拍分组范围”，展示当前数值和“更严格 / 更宽松”的含义。

## 信息架构

### 主工作台

顶栏只保留：返回、相册与组进度、评分保存状态、设置、蓝色“完成”。照片舞台、胶片带和星级评分继续占据主体。高级分组操作不再作为页面底部的临时工具条出现。

### 设置 Sheet

设置分为三个 iOS grouped-list 区段：

1. **浏览与评分**：每屏照片数量（分段控件）、评分后自动前进（Switch）。
2. **连拍分组**：连拍分组范围（滑块、实时数值、严格/宽松标签）、在当前照片前拆分、与下一组合并、撤销最近操作。
3. **说明**：解释评分会即时保存在本机，只有点击“完成”后才会写入照片元数据或复制文件。

拖动分组滑块时只更新预览值，松开或键盘确认时提交重新分组，避免连续触发昂贵请求。关闭 Sheet 后保留当前焦点与照片批次。

### 完成 Sheet

点击“完成”打开结果 Sheet，顶部用大号统计显示入选照片、已评分和总数。下方提供两个带说明的动作：

- **写入 Lightroom 星级**：进入现有预览、关闭 Lightroom 确认和安全提交流程。
- **复制入选照片**：进入现有目标目录、冲突预览和复制流程。

“返回继续选片”是次级动作。示例相册继续只允许预览，不发生文件写入。

## 视觉语言

- 使用 iOS 深色系统色：纯黑舞台、分层灰色浮层、`#0A84FF` 交互蓝色。
- Sheet 使用 20–24px 圆角、细边框、柔和阴影和背景模糊；设置项采用 grouped-list 卡片。
- 使用系统字体栈，标题采用清晰的层级和略紧字距。
- Switch、分段控件、滑块和列表行遵循至少 44px 触控尺寸。
- 动画只用于 Sheet 的淡入与轻微位移，并遵循 `prefers-reduced-motion`。
- 所有按钮使用文本或稳定的 CSS 图形配合可访问名称，不引入外部图标依赖。

## 状态与数据流

- `CullingWorkspace` 管理设置 Sheet、完成 Sheet和保存状态。
- `CullingPreferences` 继续验证并保存每屏数量与自动前进偏好；删除不再需要的 `groupToolsExpanded`。
- 评分动作开始时显示“正在保存”，成功后显示“已保存”，失败时显示“保存失败”且不自动前进。
- 分组灵敏度仍通过现有 `onSensitivity` 持久化；拆分、合并和撤销复用现有命令。
- Sheet 使用 dialog 语义，打开后聚焦关闭按钮，Escape 关闭，并将焦点还给触发按钮。

## 错误处理

- 评分失败在顶栏保存状态和现有错误区域同时呈现，不伪装为成功。
- 分组操作失败保留 Sheet 与当前输入值，用户可重试。
- 完成与导出流程继续使用现有事务、安全预览与错误提示。
- Sheet 关闭不清除未改变的浏览偏好或工作台焦点。

## 测试

- 设置按钮可打开/关闭 Sheet，Escape 可关闭，dialog 标签和可访问名称正确。
- 每屏数量、自动前进和分组范围均能在设置中操作并持久化。
- 分组拆分、合并和撤销从设置 Sheet 调用原有命令。
- 评分时依次呈现保存中与已保存；失败显示失败且焦点不前进。
- 主顶栏不再出现旧密度控件、自动前进复选框或底部“调整分组”。
- “完成”打开统计和现有导出组件；移动端无横向溢出。
- 全量单元测试、类型检查、lint、构建和 E2E 保持通过。

## 非目标

不新增云同步、账号、批量审美评分、照片删除、Lightroom 目录数据库写入或新的服务端配置模型。本次只改善现有能力的发现性、反馈和视觉一致性。
