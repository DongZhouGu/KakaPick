# 咔咔选品牌升级设计

## 目标

将产品品牌从 BurstPick 升级为“咔咔选 / KakaPick”，建立易理解、易传播、面向中外用户的品牌识别，同时保留现有产品结构、核心交互和深色沉浸式选片工作台。

这次升级解决的是品牌表达不统一，而不是重新设计产品。用户应继续使用相同的相册打开、自动分组、并排比较、评分、淘汰和导出流程；升级后，名称、Logo、颜色、文案和界面状态应呈现为同一个完整产品。

## 品牌定位

- 中文品牌名：咔咔选
- 英文品牌名：KakaPick
- 产品类别：本地摄影选片工具
- 目标用户：专业摄影师与摄影爱好者
- 核心痛点：大量照片耗费过多选片时间
- 核心承诺：帮助用户更快完成大量照片的比较、评分与导出
- 主口号：拍得多，也能选得快。
- 功能说明：连拍自动成组，好片更快看清。

“咔咔”来自连续按下快门的声音，建立摄影联想；“选”直接说明产品用途。品牌语气轻松、直接、可信，不卖萌，不将确定性算法夸大为万能 AI，也不承诺完全自动替代摄影师判断。

## 品牌叙事与宣传内容

### 一句话介绍

咔咔选是一款面向摄影师和摄影爱好者的本地选片工具，自动整理连续拍摄和相似照片，帮助用户更快比较、评分与导出。

### 首页主文案

标题：

> 拍得多，也能选得快。

说明：

> 连拍自动成组，并排看清差异，快速完成评分、淘汰与导出。

### 三项核心价值

1. 自动整理：相似照片按拍摄时间与画面特征自动成组。
2. 快速判断：并排比较、高清放大、键盘连续评分。
3. 安心本地：照片不上传、不修改 RAW，结果可交给 Lightroom。

### 长介绍

咔咔选是一款面向摄影师和摄影爱好者的本地选片工具。它自动整理连续拍摄和相似照片，让用户更快比较清晰度、表情与瞬间差异，再将评分安全导出到 Lightroom，或直接复制入选照片。照片、缩略图和评分处理均留在本机。

### 文案规则

- 优先使用短句和明确动作，例如“选择照片文件夹”“继续选片”“导出入选照片”。
- 用户可恢复的错误应说明下一步，不使用只有技术人员能理解的错误文本。
- 对自动检测使用“辅助”“标记”“建议”等词，不使用“替你选好”“绝不漏片”等绝对承诺。
- 在面向中文用户的主要界面使用“咔咔选”；KakaPick 用于英文环境、包名、文件名或中英文联合品牌场景。
- 隐私表达保持具体：不上传照片、不修改 RAW、仅在本机处理。

## Logo 系统

### 核心图形

Logo 结合两个信息：

1. 两个轻微错位、叠放的圆角取景框，代表连续拍摄、相似照片和自动分组。
2. 前方取景框中的简洁勾形，代表快速完成选择。

图形不直接绘制相机机身，不使用完整光圈叶片，也不使用拟人表情。视觉应轻快但克制，首先像可靠的摄影工具，其次才体现趣味。

### 几何原则

- 基础画布使用正方形比例。
- 取景框使用一致线宽和圆角，后方框向右上或左上偏移约一成画布宽度。
- 勾形位于前方框的视觉中心，端点使用圆头。
- 图形在 16px 下仍能分辨为叠放照片与勾选，不依赖细线、文字或渐变。
- 图形四周保留至少一个主线宽的安全区。

### Logo 组合

- 主组合：图形 + “咔咔选”中文标准字。
- 国际组合：图形 + “KakaPick”英文标准字。
- 紧凑组合：仅图形，用于 App 图标、浏览器图标和窄顶栏。
- 中英文联合组合只用于官网、发布材料和关于页面，不放入空间有限的工作台顶栏。

### App 图标

App 图标采用深色圆角方形底板、品牌橙取景框和暖白勾形。图标不得加入文字。macOS 图标需要在系统圆角遮罩内保持足够留白，并分别检查 16、32、64、128、256、512 和 1024px 尺寸。

### 单色与反色

- 深色背景：品牌橙图形或暖白单色图形。
- 浅色背景：深灰单色图形或品牌橙图形。
- 禁止给 Logo 添加投影、描边外发光、多色渐变或未经定义的状态颜色。

## 颜色体系

### 品牌色

| Token | 色值 | 用途 |
|---|---:|---|
| `brand-orange` | `#FF7A1A` | Logo、主要行动、当前焦点 |
| `brand-orange-hover` | `#FF8F3D` | 主要行动悬停 |
| `brand-orange-pressed` | `#E86100` | 主要行动按下 |
| `brand-orange-soft` | `rgba(255, 122, 26, 0.16)` | 轻量选中背景与焦点光晕 |

### 中性色与状态色

| Token | 色值 | 用途 |
|---|---:|---|
| `canvas` | `#0B0C0E` | 应用背景 |
| `surface` | `#181A1E` | 卡片、面板 |
| `surface-raised` | `#24272D` | 浮层、悬浮控件 |
| `text-primary` | `#F7F7F5` | 主文字 |
| `text-secondary` | `#A6A8AD` | 辅助文字 |
| `success` | `#35C76F` | 成功、已保存、保留状态 |
| `warning` | `#FFB547` | 警告、需注意状态 |
| `danger` | `#FF5C57` | 淘汰、错误、破坏性行动 |
| `rating` | `#FFD35A` | 星级评分 |

品牌橙只承担品牌、焦点与主要行动，不替代星级黄色、淘汰红色、成功绿色等状态语义。同一页面原则上只设置一个高强调的橙色主要按钮。

所有正文、按钮和状态文字须满足 WCAG AA 对比度要求；焦点不能只通过颜色表达，还应保留轮廓或形状变化。

## 字体与排版

- 中文使用系统无衬线字体栈，优先苹方；英文优先 SF Pro。
- 产品名采用 650 至 700 字重，正文采用 400 至 500 字重。
- 标题紧凑但不使用夸张的超大展示字；照片仍是工作台的第一视觉层级。
- 数字统计使用等宽数字特性，避免评分数和扫描进度跳动。
- 界面不使用全大写中文、装饰性手写体或拟物摄影字体。

## 界面更新范围

本次只进行品牌层统一，不改变信息架构、页面流、快捷键、照片布局、评分模型、自动分组算法或导出行为。

### 欢迎页

- 使用主 Logo 组合替换现有临时品牌标记。
- 品牌名改为“咔咔选”。
- 主标题改为“拍得多，也能选得快。”
- 说明改为“连拍自动成组，并排看清差异，快速完成评分、淘汰与导出。”
- 保留最近相册、选择文件夹、示例相册、手动路径和隐私说明的现有结构。
- 主按钮使用品牌橙；次按钮保持中性层级。

### 扫描与空状态

- 扫描页显示紧凑图形 Logo，并沿用清晰的阶段名称和进度。
- 语气简洁，例如“正在整理相似照片”，不加入无依据的剩余时间。
- 空状态与错误状态使用同一套按钮、圆角和文字层级。

### 选片工作台

- 顶栏将 BurstPick 替换为紧凑图形和“咔咔选”。
- 保留现有深色沉浸式布局、照片舞台、胶片带、筛选、设置和导出位置。
- 当前照片焦点由旧金色统一为品牌橙；焦点仍同时使用边框和光晕。
- 星级继续使用评分黄，淘汰继续使用危险红，保留状态继续使用成功绿。
- “导出评分…”可根据实际内容统一为“完成选片”或“导出入选照片”；具体实现应以当前导出面板入口语义为准，并同步更新测试。

### 设置、完成与导出面板

- 统一面板圆角、边框、内边距、标题层级和底部行动区。
- 每个面板只保留一个橙色主要行动；取消、返回和辅助操作使用中性样式。
- 成功、警告、失败结果使用对应状态色，不使用品牌橙冒充状态。
- 不重排现有导出路径，也不改变 Lightroom 与直接复制的安全确认流程。

## 设计 Token 与组件规则

实施时应在 CSS 根级统一定义品牌、表面、文字、状态、圆角、间距、阴影和动效 token，逐步替换当前分散的硬编码颜色。此次只替换本次涉及组件的视觉值，不进行与品牌升级无关的 CSS 重构。

建议统一以下组件：

- Logo：`BrandMark` 与 `BrandLockup`
- 主要、次要和危险按钮
- 表单输入与焦点状态
- 面板与浮层容器
- 状态提示与错误横幅
- 空状态与加载状态

组件必须继续使用原生语义、可见焦点和现有可访问名称。装饰性 Logo 图形应对辅助技术隐藏；带返回行为的品牌按钮保留明确的 `aria-label`。

## 动效原则

- Logo 可在启动或扫描时使用一次轻微的“双帧错位归位”动效，表达咔咔连拍。
- 持续工作界面不循环播放品牌动效，避免干扰照片判断。
- 按钮和焦点过渡保持在 120 至 200ms。
- `prefers-reduced-motion` 下禁用位移动效，仅保留即时状态变化。

## 命名迁移边界

用户可见名称统一改为“咔咔选”或“KakaPick”，包括页面标题、应用显示名、欢迎页、顶栏、README 和发布文案。

内部技术标识在本次不强制迁移：

- npm 包名、目录名、API token header、sessionStorage key 和应用数据目录可继续使用 `burstpick` / `BurstPick`。
- 保留内部标识可避免会话、偏好和已安装本地数据无意失效。
- 若未来正式发布需要迁移 bundle identifier 或应用数据目录，应单独设计兼容迁移，不在品牌视觉升级中顺带更改。

## 资产与文档交付

- 主 Logo 的可编辑矢量源文件。
- 深色、浅色和单色 SVG 导出。
- macOS 1024px App 图标源图及所需尺寸导出。
- 浏览器 favicon。
- README 中的产品名、定位、主口号和功能介绍。
- 本设计规格及后续实施计划。

## 测试与验收

### 自动验证

- 现有组件、单元、类型、lint 和构建测试继续通过。
- 更新依赖旧产品名或旧欢迎文案的组件测试。
- 为 Logo 组件验证装饰语义、品牌文字和返回按钮可访问名称。
- 对主题 token 和关键状态类做必要的组件级断言，不对像素颜色进行脆弱快照测试。

### 视觉验收

- 在欢迎页、扫描页、工作台、设置、完成和导出状态检查品牌一致性。
- 检查 1440px、760px 与 390px 宽度。
- 检查 16px 顶栏图标、macOS Dock 图标和浅深背景下的 Logo 可读性。
- 检查默认、悬停、键盘焦点、按下、禁用、保存成功、警告和错误状态。
- 检查照片颜色不受品牌滤镜、混合模式或背景溢色影响。

### 验收标准

1. 所有主要用户可见位置不再出现 BurstPick 品牌名。
2. 用户首次看到欢迎页即可理解这是帮助快速完成大量照片筛选的本地工具。
3. Logo 在 16px 和单色条件下仍可识别。
4. 品牌橙、评分黄、淘汰红和成功绿职责清晰，没有混用。
5. 现有选片、评分、分组、设置和导出流程没有行为变化。
6. 品牌升级后页面、App 图标和项目文档使用相同名称、口号和视觉规则。

## 非目标

- 不重做工作台信息架构或交互流程。
- 不改变自动分组、评分、淘汰、保存和导出逻辑。
- 不新增账号、云同步、订阅或协作功能。
- 不在本次迁移内部存储 key、应用数据目录或 API header。
- 不以生成式插画或复杂吉祥物作为主要品牌资产。
