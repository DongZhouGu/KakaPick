## 文档权威顺序

同一主题出现差异时，按以下顺序判断：

1. 当前代码、自动化测试与公开 API 合同；
2. `docs/` 顶层关键文档；
3. 根目录 `README.md` 中的项目概览与使用说明。

发现关键文档与代码不一致时，应在同一次变更中更新文档。

## 快速阅读路径

- 第一次了解项目：产品说明 → 品牌规范 → UI 设计体系。
- 准备修改功能：产品说明 → 系统架构 → 开发指南。
- 准备发布：验证与发布 → 根目录 `README.md`。
- 了解版本变化：发布历史 → 对应版本说明。
