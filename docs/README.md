# 咔咔选项目文档

这里是咔咔选（KakaPick）的项目知识入口。顶层文档描述产品当前形态，是设计、开发和发布时优先参考的权威来源；`superpowers/` 保存按时间形成的设计规格与实施计划，用于追溯背景和决策过程。

## 关键文档

| 文档 | 适合谁 | 内容 |
|---|---|---|
| [产品说明](product.md) | 产品、设计、研发 | 定位、用户、核心问题、工作流、范围与成功标准 |
| [系统架构](architecture.md) | 研发、维护者 | 模块边界、数据流、存储、安全、分组和导出架构 |
| [品牌规范](brand.md) | 产品、设计、市场 | 名称、定位、口号、Logo、颜色、文案和品牌资产 |
| [UI 设计体系](design-system.md) | 设计、前端 | 设计原则、Token、组件、状态、响应式与无障碍规则 |

## 文档权威顺序

同一主题出现差异时，按以下顺序判断：

1. 当前代码、自动化测试与公开 API 合同；
2. `docs/` 顶层关键文档；
3. `docs/superpowers/specs/` 中已确认的设计规格；
4. `docs/superpowers/plans/` 中的实施计划。

实施计划描述“当时准备怎么做”，不保证每个细节仍代表当前实现。发现关键文档与代码不一致时，应在同一次变更中更新文档。

## 快速阅读路径

- 第一次了解项目：产品说明 → 品牌规范 → UI 设计体系。
- 准备修改功能：产品说明 → 系统架构 → 开发指南。
- 准备发布：验证与发布 → 根目录 `README.md`。
- 调查历史决策：设计与实施记录 → 对应专项规格。
