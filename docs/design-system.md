# UI 设计体系

## 设计目标

咔咔选的界面服务于长时间、高密度的照片判断。照片是第一视觉层级，应用控件保持紧凑但可辨认；品牌带来一致性，不应抢夺照片注意力。

## 设计原则

1. **照片优先**：深色中性背景，不给照片添加滤镜、混合模式或环境染色。
2. **状态清楚**：焦点、评分、保留、淘汰、保存和错误使用稳定且互不混淆的语义。
3. **高频高效**：键盘连续操作无额外确认，指针操作也能完成同样任务。
4. **逐层展开**：工作台展示常用操作，高级分组和偏好收进设置面板。
5. **克制品牌化**：橙色只用于品牌、焦点和主要行动。

## 设计 Token

当前 Token 定义在 `src/client/styles.css` 的 `:root`。新增视觉值应优先复用 Token，不在组件中散落新的十六进制颜色。

### 色彩职责

- 品牌与焦点：`--brand-orange`
- 星级：`--rating`
- 保留和保存成功：`--success`
- 警告：`--warning`
- 淘汰、错误和危险操作：`--danger`
- 画布和层级：`--canvas`、`--surface`、`--surface-raised`
- 文字：`--text-primary`、`--text-secondary`

### 字体

中文优先使用苹方系统字体，英文优先使用 SF Pro。正文保持 400–500 字重，标题和产品名使用 650–700。文件名和统计数字应优先保证清晰，不使用装饰字体。

### 圆角与层级

- 小控件：6–10px；
- 普通按钮和输入：10–12px；
- 卡片：14–18px；
- Sheet 和完成面板：20–24px。

阴影只用于浮层、当前卡片或需要与照片背景分离的元素，不用阴影表达所有边界。

## 核心页面

### 欢迎页

展示品牌、核心价值、最近相册和打开入口。主行动只有“选择照片文件夹”；示例相册和手动路径保持次级。隐私说明始终可见。

### 扫描与空状态

显示真实阶段和进度，不伪造剩余时间。错误提供可执行的下一步；空相册说明支持的文件类型并允许重新选择。

### 沉浸式工作台

顶栏承担返回、组导航、保存状态、筛选、设置和完成入口。主区域只保留照片舞台、胶片带和简短快捷键提示。照片焦点使用橙色边框与光晕，不能只靠颜色区分。

### 设置与完成面板

设置按浏览、辅助检测、分组、快捷键和数据分区。完成面板先总结入选结果，再提供直接复制和 Lightroom 两条明确路径。一个面板只设置一个最高强调的橙色行动。

## 组件规则

- **BrandMark**：装饰图形对辅助技术隐藏；作为返回按钮时提供明确 `aria-label`。
- **按钮**：主要按钮用品牌橙，次要按钮用中性表面，危险按钮用红色且只出现在明确危险操作。
- **照片卡片**：文件名、配对状态、星级和辅助问题标签保持固定位置。
- **状态提示**：保存中、已保存、失败和撤销使用文字与颜色共同表达。
- **筛选器**：选中状态使用形状或表面变化，不只改变文字颜色。
- **对话框与 Sheet**：具有标题、明确关闭入口、焦点管理和 Esc 退出。

## 动效

按钮、焦点和面板过渡保持 120–200ms。品牌 Logo 只可在启动或扫描时使用一次轻微的双帧归位；工作台不循环播放装饰动画。`prefers-reduced-motion` 下禁用位移动画。

## 响应式

- 桌面是主要体验，优化目标为常见笔记本和外接显示器。
- 760px 以下顶栏允许换行，筛选器占满一行，照片舞台保持可操作。
- 390px 以下欢迎页、输入与完成面板使用单列。
- 触控目标应达到 44×44px；若视觉图标较小，使用透明点击区域补足。

## 无障碍

- 正文和控件满足 WCAG AA 对比度。
- 所有操作可用键盘完成，并显示可见焦点。
- 装饰图片使用空替代文本，信息图片提供可理解名称。
- 保存、扫描和导出进度通过 `role="status"` 或等效 live region 宣告。
- 不在输入、范围控件或对话框中劫持选片快捷键。
